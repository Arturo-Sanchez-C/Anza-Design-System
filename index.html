Workflow how everything connects

The system has three layers. Set them up once — then anyone on the team can create consistent artifacts without any manual setup per session.

1
Join the right Claude Project — already set up

Two shared projects are ready: Anza Design System for UI and component work, Solar for domain-specific tools. Open the project, start a new chat inside it, and Claude already knows the full design system.

2
Create artifacts inside the project — anyone on the team

Open a new chat inside the Anza Project and describe what you want. Reference Figma via MCP for specific components if needed. Claude already knows the design system.

3
Save to the Notion repository — after each artifact

Screenshot the artifact, export the code, and add an entry to the Anza Artifacts Notion database so the team can browse, preview, and reuse it.

💡
No more pasting. Once the Claude Project is set up and shared, the design system is always active. The only thing each person needs to do is open a new chat inside the project.

05

Best practices write prompts that work

✓ Do
  • Give business context: what it shows, who it's for
  • Name the component type: card, table, form, dashboard
  • Specify variants when known (Button variant=Destructive)
  • Use React for interactive artifacts, HTML for static pages
✕ Don't
  • Use hex colors not in the token list
  • Request fonts other than Inter
  • Skip the system prompt and ask for visual styles only
  • Mix in Material UI, Ant Design, or other libraries

Prompt template

Template
# What it is
Create a [card / table / form / dashboard] for [audience].

# What it shows
Display [data/content]. The user can [action].

# Components (optional)
Use [Component] for [part], [Component] for [part].

# Output
Generate as [React / HTML].

When to use each output type

HTML
Static pages, docs, guides, emails

No state. Easy to share as a file.

React
Dashboards, interactive forms, components with state

Filters, tabs, modals, hooks. Compatible with Anza's codebase.

SVG
Diagrams, flows, architecture maps

Lightweight, exportable into Figma.

07

Keeping it current

New component added to Figma

Ask Claude to re-query just the components section from Figma and update the prompt. Re-download the MD.

Color or typography tokens changed

New conversation → query Figma Foundations library → update and re-download the prompt.

Who owns the prompt

The design team — they control the Figma libraries and can validate the prompt reflects the current system.

📌
The design team owns the prompt. Store the latest version in the Anza Claude Project knowledge base and in Notion with a "last updated" date. When an artifact looks off, that's the signal to update the Project Instructions and re-upload the file.

Submission flow — after finishing an artifact

1
Screenshot the rendered artifact

Right-click the artifact panel in Claude → Save image. This becomes the preview card in the Notion gallery.

2
Export the code

Click the copy icon on the artifact → paste into a .jsx or .html file named after the artifact.

3
Add a new entry to the Notion database

Open the Anza Artifacts database → New → fill in the properties above. Upload the screenshot to Preview, the code file to Code, and paste the Claude project conversation URL.

✓
Done — it's in the gallery

Anyone on the team can find it by type, author, or team. They can see the preview and copy the code directly from Notion without rebuilding.

Recommended Notion views

Gallery — browse by type

Group by Type. Show Preview as card cover. Default view for exploring.

Table — my artifacts

Filter by Author = me. Shows your own artifacts with status and last updated.

Board — by status

Columns: Draft / Ready / Deprecated. Good for design team review.

Table — by team

Filter by Team. See what Design, Engineering, and Product have each built.

📌
Connect them: Add the Notion database URL to the Anza Claude Project knowledge base. That way Claude can answer "has this been built before?" and link to the existing artifact before someone rebuilds it from scratch.
